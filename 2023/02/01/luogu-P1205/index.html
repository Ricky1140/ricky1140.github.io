洛谷P1205 Java解法

2023-02-01 数据结构与算法 洛谷 922 字

[P1205 USACO1.2] 方块转换 Transformations - 洛谷 | 计算机科学教育新生态 (luogu.com.cn)

这个题其实画了图就好写了！

以输入输出样例为例，我们的初始方块为：

​ @ - @

​ - - -

​ @ @ -

转换完的方块为：

​ @ - @

​ @ - -

​ - - @

肉眼可见的旋转 90°

如果是在纸上画的图可以转纸，电脑上画的就转脖子吧

但在程序里转换我们怎么换呢？

  • 我们的数组是从左到右，从上到下

  • 旋转 90° 后是从下到上，从左到右

对图形不敏感的也可以列数组位置：

假设旋转前的方块记为 a[][] ，旋转后的方块记为 t[][]

那么：

t[0][0] = a[2][0];
t[0][1] = a[1][0];
t[0][2] = a[0][0];

t[1][0] = a[2][1];
t[1][1] = a[1][1];
t[1][2] = a[0][1];

t[2][0] = a[2][2];
t[2][1] = a[1][2];
t[2][2] = a[0][2];

这应该能看出来了吧！

那么步骤 2 步骤 3 的旋转就是多调用几次旋转 90°

接下来的步骤 4 还请看清楚再写，我就因为看错题了 wa 了（悲）

按垂直线进行镜像，也可以画图，我懒得在电脑上敲图了这里直接写数组位置：

t[0][0] = a[0][2];
t[0][1] = a[0][1];
t[0][2] = a[0][0];

t[1][0] = a[1][2];
t[1][1] = a[1][1];
t[1][2] = a[1][0];

t[2][0] = a[2][2];
t[2][1] = a[2][1];
t[2][2] = a[2][0];

步骤 5 就是组合以上步骤，步骤 6 就是直接对比两个方块是否相等，当以上操作都没有用的时候就执行步骤 7。

那么思路总结完成！

以下是踩坑记录！

数组之间的对比不能用 equals() ，还没看原理，大概跟地址数值引用之类的概念有关系？

以及最开始写旋转函数的时候懒得写初始化直接给等于传进去的数组了，导致地址一样，过渡数组的变化也影响到了本身。

以下是源代码：

import java.util.Scanner;

public class P1205 {
	public static void main(String[] args) {
		Scanner sc = new Scanner(System.in);
		
		int n = sc.nextInt();	
		
		String[] a = new String[n+1];
		String[] b = new String[n+1];
		
		char[][] c = new char[n][n];
		char[][] d = new char[n][n];
		
		for (int i = 0; i <= n; i++) {
			a[i] = sc.nextLine();
		}
		
		for (int i = 0; i < n; i++) {
			b[i] = sc.nextLine();
		}
		
		for (int i = 0; i < n; i++) {
			c[i] = a[i + 1].toCharArray();
			d[i] = b[i].toCharArray();
		}
		
		if (identical(d, typeOne(c))) {
			System.out.println("1");
		} else if (identical(d, typeTwo(c))) {
			System.out.println("2");
		} else if (identical(d, typeThr(c))) {
			System.out.println("3");
		} else if (identical(d, typeFour(c))) {
			System.out.println("4");
		} else if (identical(d, c)) {
			System.out.println("6");
		} else {
			char[][] temp = typeFour(c);
			if (identical(d,typeOne(temp)) || identical(d,typeTwo(temp)) || identical(d,typeThr(temp))) {
				System.out.println("5");
			} else {
				System.out.println("7");
			}
		}
		
		sc.close();
	}
	
	public static char[][] typeOne(char[][] a) {
		int n = a.length;
		char[][] temp = new char[n][n];
		
		for (int i = 0; i < n; i++) {
			for (int j = 0; j < n; j++) {
				temp[i][j] = a[a.length - j - 1][i];
			}
		}
		
		return temp;
	}
	
	public static char[][] typeTwo(char[][] a) {
		int n = a.length;
		char[][] temp = new char[n][n];
		
		temp = typeOne(a);
		temp = typeOne(temp);
		
		return temp;
	}
	
	public static char[][] typeThr(char[][] a) {
		int n = a.length;
		char[][] temp = new char[n][n];
		
		temp = typeOne(a);
		temp = typeOne(temp);
		temp = typeOne(temp);
		
		return temp;
	}
	
	public static char[][] typeFour(char[][] a) {
		int n = a.length;
		char[][] temp = new char[n][n];
		
		for (int i = 0; i < n; i++) {
			for (int j = 0; j < n; j++) {
				temp[i][j] = a[i][n - j - 1];
			}
		}
		
		return temp;
	}
	
	public static boolean identical(char[][] a, char[][] b) {
		boolean flag = true;
		
		for (int i = 0; i < a.length; i++) {
			for (int j = 0; j < a[0].length; j++) {
				if (a[i][j] != b[i][j]) {
					flag = false;
					break;
				}
			}
		}
		return flag;
	}
}
