    创建虚拟机

    VMware Workstation 17.0 Pro在win10系统的安装

    官网链接：Download VMware Workstation Pro

    下载安装包后，双击，按照自己需求更改即可，密钥可上网搜索，有很多。
    首先创建一个空的虚拟机
    打开VMware17软件后，点击创建新的虚拟机，**勾选自定义(高级)**：

    • 虚拟机硬件兼容性：按自己需求（可默认）
    • 安装客户机操作系统：勾选稍后安装操作系统，
    • 选择客户机操作系统：勾选Linux，
    • 命名虚拟机：命名随便（可默认），自定义安装位置，
    • 处理器配置：根据自己电脑的配置来分配（可默认）
    • 虚拟机的内存：根据自己电脑的配置来分配（可默认）
    • 网络类型：勾选使用桥接网络，
    • 选择I/O控制器类型，选择磁盘类型：直接选推荐（默认），
    • 选择磁盘：勾选创建新虚拟磁盘（默认），
    • 指定磁盘容量：按自己需求（可默认），可勾选立即分配所有空间，勾选将虚拟磁盘拆分成多个文件（默认），
    • 指定磁盘文件：命名随便（可默认），
    • 已准备好创建虚拟机：可以在自定义硬件查看之前的配置检查修改，点击完成，就创建好一个空的虚拟机了。

    之后添加操作系统：在空虚拟机左侧，点击虚拟机设置，点击CD/DVD(SATA)，点击使用ISO映像文件，在浏览中找到之前下好的映像文件，点击确定，就可以开启虚拟机使用了。

    下载并安装 Ubuntu

    改自VMware17虚拟机安装Ubuntu最新版本(Ubuntu22.04LTS)详细步骤_wkd_007的博客-CSDN博客

    Ubuntu 22.04.1 LTS在VMware虚拟机的安装，LTS意为“长期支持”，一般为5年。

    官网链接： Ubuntu 22.04.1 LTS (Jammy Jellyfish)

    下载 ubuntu-22.04.1-desktop-amd64.iso

    在 VMware 打开虚拟机，不要启动，在虚拟机的选项卡上点击鼠标右键，依次选择 设置 -> 硬件 -> CD/DVD(SATA)，勾选使用ISO映像文件，选择前面下载好的 ubuntu-22.04.1-desktop-amd64.iso 镜像文件，点击确定

    现在点击开启此虚拟机，等待进入 GRUB 界面，然后选择 Try or Install Ubuntu，按 Enter，等待

    安装界面

    进入过后会显示安装界面，不擅长英文的人可以在左边栏往下拉选择 中文（简体），然后点击 安装 Ubuntu

    键盘布局

    左边选择 Chinese，右边选择 Chinese - Hanyu Pinyin(with AltGr dead keys)

    更新和其他软件

    • 勾选最小安装
    • 勾选安装 Ubuntu 时下载更新

    安装类型

    勾选清除整个磁盘并安装

    弹窗时选继续

    您在什么地方？

    选择 Shanghai

    您是谁？

    填写自己用户名，如果是自己玩建议勾个自动登录

    继续，重启

    测试网络

    在命令行中 ping 一下百度看看有没有网

    没有的话移步踩坑，还没的话自行搜索

    ping baidu.com

    Hadoop伪分布式搭建

    修改自Hadoop安装搭建伪分布式教程（全面）吐血整理_hadoop伪分布式搭建_剁椒沙丁鱼的博客-CSDN博客

    创建hadoop用户

    因为还有其他要部署的，所以建议创个用户

    Ctrl + Alt + T 进入命令行模式

    输入以下语句，创建一个名为hadoop的用户，useradd命令的-m用来指定用户名，-s用来指定用户登录时所需的shell文件

    sudo useradd -m hadoop -s /bin/bash

    由于部门权限规定或安全限制，负责部署hadoop的管理员没有linux root权限，但按照最佳做法，安装时有一些操作需要以root用户身份执行。以下给予该用户root权限：

    sudo adduser hadoop sudo

    更新命令

    由于ubuntu系统刚完成安装，需要更新相关一些软件，确保接下来的操作可以正常完成

    sudo apt-get update

    安装 vim 编辑器，跳出来的提示直接回复 Y

    sudo apt-get install vim

    配置SSH

    安装 SSH server

    sudo apt-get install openssh-server

    连接本地 SSH，按提示输入 yes，输入 hadoop 用户的密码进行登录

    ssh localhost

    设置无密码登录

    退出连接

    exit

    进入 SSH 对应目录

    cd ~/.ssh/

    输入生成 SSH 私钥与公钥的命令，-t用于声明密钥的加密类型，输入 Hadoop 密码。这一步会提醒设置 SSH 密码，输入密码时直接回车就表示无密码，第二次输入密码回车确定，最后一次提交

    ssh-keygen -t rsa

    将生成的SSH的公钥加入目标机器的SSH目录下，这里采用cat命令与>>，cat file1>>file2的含义为将file1内容append到file2中

    cat ./id_rsa.pub >> ./authorized_keys

    安装 Java环境

    下载 Java

    sudo apt-get install default-jre default-jdk

    配置环境变量

    通过vim编辑器打开环境变量的设置文件

    sudo vim ~/.bashrc

    按 i 进入 insert 模式，在文件尾部加上以下语句

    export JAVA_HOME=/usr/lib/jvm/default-java
    export HADOOP_HOME=/usr/local/hadoop
    export PATH=$HADOOP_HOME/bin:$HADOOP_HOME/sbin:$PATH

    按 esc 退出，输入:wq保存并退出（强制执行为前面加感叹号）

    使环境变量生效

    source ~/.bashrc

    下载并配置 Hadoop 环境

    下载并解压

    在虚拟机中的火狐浏览器进入清华大学的镜像网站（官网太慢）Index of /apache/hadoop/common/stable (tsinghua.edu.cn)，下载 hadoop-3.3.5.tar.gz，默认保存至 downloads 文件夹中

    下载其他版本也行但得是 .tar.gz

    如果在安装 Ubuntu 时使用的是中文，则保存在在下载文件夹中

    进入 downloads 文件夹，右键属性查看压缩包的绝对路径，再解压到 /usr/local 目录

    是英文就把下载改成 downloads

    下的是哪个版本就解压什么版本

    sudo tar -zxf /home/hadoop/下载/hadoop-3.3-5.tar.gz -C /usr/local

    进入刚刚解压后存放的目录下

    cd /usr/local

    将该文件夹重命名为 hadoop，方便后续操作

    sudo mv ./hadoop-3.3.5/ ./hadoop

    配置环境

    将已重命名的该文件夹的拥有者指定给用户 hadoop，否则权限不足

    sudo chown -R hadoop ./hadoop

    此时虽然 jre 已经配置好，但是运行 hadoop 的时候还是有找不到 jdk 的情况出现，所以还需要在 hadoop 的环境文件中进行修改

    进入目录

    cd ./hadoop/etc/hadoop

    使用 vim 编辑器，打开环境变量文件

    sudo vim hadoop-env.sh

    按 i，添加以下语句

    export JAVA_HOME=/usr/lib/jvm/default-java

    按 esc 退出，输入 :wq 保存并退出，然后使环境变量生效

    source hadoop-env.sh

    现在，hadoop 搭建完毕，可以通过调用版本号测试 hadoop 是否可用

    cd /usr/local/hadoop
    ./bin/hadoop version

    如果搭建成功就会显示 hadoop 的版本号等信息

    配置四个文件

    此时需要在 hadoop 文件夹下的 hadoop 文件中

    绝对路径：cd /usr/local/hadoop/etc/hadoop

    core-site.xml

    先打开xml文件，注意 ./ 表示当前文件夹下，当前应该在第二个hadoop文件夹下cd ./etc/hadoop/

    sudo gedit core-site.xml

    打开xml文件应该是有内容的，如果没有内容说明打开文件不对，需要检查输入的路径及文件名是否正确。将替换为以下内容，第一个属性表示Hadoop重要临时文件的存放目录，指定后需将使用到的所有子级文件夹都要手动创建出来，否则无法正常启动服务；第二个被官方称之为默认系统文件的名称（the name of the default file system），实际上决定了文件系统的主机号、端口号，对于伪分布式模型来说，其主机地址为localhost。

    <configuration>
    	<property>
    		<name>hadoop.tmp.dir</name>
    		<value>file:/usr/local/hadoop/tmp</value>
    		<description>Abase for other temporary directories.</description>
    	</property>
    	<property>
    		<name>fs.defaultFS</name>
    		<value>hdfs://localhost:9000</value>
    	</property>
    </configuration>

    hdfs-site.xml

    进入 hdfs-site.xml

    sudo gedit hdfs-site.xml

    对hdfs-site.xml进行同样的替换操作，属性的含义分别为复制的块的数量、DFS管理节点的本地存储路径、DFS数据节点的本地存储路径

    <configuration>
    	<property>
    		<name>dfs.replication</name>
    		<value>1</value>
    	</property>
    	<property>
    		<name>dfs.namenode.name.dir</name>
    		<value>file:/usr/local/hadoop/tmp/dfs/name</value>
    	</property>
    	<property>
    		<name>dfs.datanode.data.dir</name>
    		<value>file:/usr/local/hadoop/tmp/dfs/data</value>
    	</property>
    </configuration>

    mapred-site.xml

    Hadoop2.X并没有像1.X提供mapred-site.xml文件，需要将样本文件复制为可使用的配置文件

    3.X 不知道，但按着 2.X 做了

    mv mapred-site.xml.template mapred-site.xml

    打开

    sudo gedit mapred-site.xml

    这里使用 yarn。具体配置内容如下

    yarn 是一种资源管理和作业调度技术，作为Hadoop 的核心组件之一，负责将系统资源分配给在 Hadoop 集群中运行的各种应用程序，并调度要在不同集群节点上执行的任务，其基本思想是将资源管理和作业调度/监视的功能分解为单独的 daemon，总体上yarn是 master/slave 结构，在整个资源管理框架中，ResourceManager 为 master，NodeManager 是 slaver。

    <configuration>
    	<property>
    		<name>mapreduce.framework.name</name>
    		<value>yarn</value>
    	</property>
    </configuration>

    yarn-site.xml

    打开

    sodu gedit yarn-site.xml

    配置yarn-site.xml文件，这里修改NodeManager上运行的附属服务即可

    <configuration>
    	<property>
    		<name>yarn.nodemanager.aux-services</name>
    		<value>mapreduce_shuffle</value>
    	</property>
    </configuration>

    启动

    格式化并启动

    为了保险起见，执行这一步前关闭命令行模式，重新打开，放弃当前路径。
    更改配置并保存后，格式化HDFS的NameNode，在这一步之前，如果hdfs-site.xml中dfs.namenode.name.dir属性指定的目录不存在，格式化命令会自动创建之；如果存在，请确保其权限设置正确，此时格式操作会清除其内部所有的数据并重新建立一个新的文件系统

    /usr/local/hadoop/bin/hdfs namenode -format

    成功格式化后启动全部进程

    start-all.sh

    成功启动后通过浏览器访问 http://localhost:9870

    如果下载的是 2.X 版本，访问 http://localhost:50070

    在命令行输入 jps 查看到六个进程

    如果缺进程就移步到后面的踩坑环节

    在 HDFS 中创建用户目录

    hdfs dfs -mkdir -p /user/hadoop

    创建 input 目录，用于输入数据

    hdfs dfs -mkdir input

    将所需的 xml 配置文件复制到 input 中（注意路径）

    hdfs dfs -put usr/local/hadoop/etc/hadoop/*.xml input

    如果系统之前运行过hadoop，则需要删除 output 目录

    warning不用在意

    hdfs dfs -rm -r output

    调用实例

    进入 hadoop 的文件夹下

    cd /usr/local/hadoop

    输入

    hadoop jar ./share/hadoop/mapreduce/hadoop-mapreduce-examples-*.jar wordcount input output

    有结果后进入 hadoop 下的 bin

    cd /usr/local/hadoop/bin

    查看运行结果

    hdfs dfs -ls ./output
    hdfs dfs -cat output/*

    到这里搭建并运行实例结束。

    踩坑

    Ubuntu 连不上网

    关闭虚拟机，打开 VMware 的编辑 -> 虚拟机网络编辑器 -> 更改设置 ->给管理员权限 -> 还原默认设置，再重新打开虚拟机

    如果还不行就自行搜索吧

    Hadoop 下载很慢

    一定要去清华大学的镜像站啊！！

    Hadoop 启动后进程不全

    关闭所有进程

    stop-all.sh

    删除 hadoop 两个目录

    rm -rf /usr/local/hadoop/logs
    rm -rf /usr/local/hadoop/tmp

    重新格式化和启动 hadoop

    hadoop namenode -format
    start-all.sh

    记得看看自己下的啥版本

    hadoop 3.X

    http://localhost:50070
